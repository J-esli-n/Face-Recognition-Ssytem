# Face Recognition Python Project

This repository contains a simple face recognition system built with Python. It uses a local image dataset to train face embeddings and then performs real-time face recognition using a webcam.

## Tech Stack

- Python 3.11 (project environment indicates a Python 3.11 virtual environment)
- OpenCV (`cv2`) for image processing, webcam capture, and display
- `face_recognition` library for face detection and facial embeddings
- `dlib` as the underlying face recognition engine used by `face_recognition`
- `imutils` for convenient image path handling
- `pickle` for saving and loading the encoded face data
- `numpy` indirectly via OpenCV and face recognition operations

## Project Structure

- `extract.py` - reads images from the `Images/` folder, detects faces, computes face embeddings, and saves them to a file named `face_enc`.
- `webcam.py` - loads saved face encodings, captures video from the connected webcam, detects faces, compares them to known embeddings, and displays the recognized names on the screen.
- `Images/` - contains folders representing different people. Each person folder stores sample face images used for training.
- `face_enc` - binary pickle file containing the serialized face encodings and names. It is generated by `extract.py`.
- `face_recognition_env/` - a local Python virtual environment with installed dependencies.

## Modules and Libraries

The project uses the following Python modules and libraries:

- `face_recognition` - provides easy-to-use face detection, encoding, and matching.
- `dlib` - supplies the underlying facial landmark and embedding models required by `face_recognition`.
- `cv2` from OpenCV - handles image loading, conversion between color spaces, face bounding box drawing, webcam capture, and window display.
- `imutils.paths` - simplifies gathering image file paths from nested folders.
- `pickle` - serializes the learned face encodings to disk and loads them back at runtime.
- `os` - builds file paths and extracts person names from image directories.
- `time` - included in `webcam.py` for possible timing operations, though the current script focuses on live capture.

## Project Flow

1. `extract.py` is run first. It loads each JPEG image listed in the script, converts the image from BGR to RGB, and uses `face_recognition.face_locations()` to locate any faces.
2. For each detected face, the script computes an embedding vector with `face_recognition.face_encodings()` and associates it with the person name derived from the image folder.
3. All face embeddings and names are stored in a dictionary and written to disk as a pickle file called `face_enc`.
4. Next, `webcam.py` loads the saved face encoding file and starts webcam video capture.
5. Each video frame is converted to grayscale for face detection using OpenCV’s Haar cascade classifier and to RGB for computing face embeddings.
6. For each face found in the frame, the script compares the live encoding to known encodings using `face_recognition.compare_faces()` and picks the best match.
7. The recognized person’s name is displayed with a rectangle drawn around each face in the camera feed.

## Usage

1. Activate the virtual environment: `face_recognition_env\Scripts\activate.bat`
2. Run `extract.py` to generate or update the `face_enc` file.
3. Run `webcam.py` to launch the live recognition demo.

## Notes

- This project is designed for local experimentation and face recognition learning.
- To expand the dataset, add more person folders and image files under `Images/`.
- Ensure the webcam is connected before running `webcam.py`.
